# 폼다 네이버 블로그 글 품질 기준 (v1.0, 2026-07-14)

제이퍼 `blog-quality.md`를 폼다 문서 도메인 + 네이버 발행 형식에 이식.
발행 전 아래를 모두 통과해야 함. 목표는 "최대 품질" - 대량생산 티가 나면 네이버 C-Rank/DIA·AI 필터에 강등되므로 밀도·다양화가 생명.

> ⚠️ 네이버라서 제외(제이퍼 site 전용): JSON-LD 3종·카카오 애드핏 슬롯·detail-shell guides·sitemap/rss·카테고리 CSS·head 메타. 네이버 에디터가 자체 처리하므로 신경 쓰지 않는다.

---

## 1. 분량·형식 (네이버)
- **본문 2,500자 이상 권장**(최소 2,000, thin 방지). v1.0 "2,000자+"를 품질 위해 상향. 단 네이버 모바일 가독 고려해 3,500자 넘기면 정보 밀도 재점검(패딩 금지).
- 제목 **28~30자**, 네이버 타겟 키워드를 **제목 앞부분**에 그대로 + 연식(2026) 유리
- **■ 소제목**으로 구획, **~해요체**, 태그 **8개**
- 표·굵게는 마크다운 금지 → 네이버 에디터 기능 사용
- em대시(—) 금지 하이픈(-) 사용 / 하이픈으로 시작하는 리스트 금지

## 2. 필수 콘텐츠 요소 (제이퍼 이식, 문서 도메인으로 치환)
- **문서 작성 실사례 3~4개** (인물·상황 기반, 추상 설명 금지)
  - 예) "34세 프리랜서 김OO씨가 3개월치 대금 420만원을 못 받았을 때, 내용증명에 이 3가지를 넣었더니…"
  - 금액·조건 다른 **대조 사례 2개 이상** (예: 이자 약정 있는 차용증 vs 없는 차용증)
  - "그래서 뭐가 달라지는데"까지 연결 (효력·불이익·절세액 등 의사결정 임팩트 1~2문장)
- **비교·기준표 1~2개** (실제 항목이 든 데이터 표. 단순 설명표 금지)
  - 예) 문서 종류별 언제 쓰나 / 상황별 필수 기재사항 / 공증 유무 차이
- **반전·함정 섹션 1개 필수** (공유·체류 유발)
  - 예) "차용증 썼는데도 효력 없는 경우", "이자 연 24%로 적었다가 초과분 전부 무효", "사직서 냈다고 바로 퇴사되는 게 아니다"
  - 반드시 법적 근거·수치 동반
- **FAQ 4개** (개수는 직전 2편과 다르게 가변 3~5), 각 답변 **4문장 이상**, 엣지케이스·예외 위주, 일반론("~일 수 있습니다") 금지. 마지막 FAQ는 폼다 도구와 자연 연결
- **신뢰 시그널 2개 이상** (법조문·고시·공식통계 중). reference_formda_legal_tax_figures 검증수치 사용:
  - 이자제한법 제2조 연 20% / 민법 제162조 소멸시효 10년 / 민법 제660조 사직 효력 / 민법 제600조 무이자 원칙 / 상증세법 가족차용 4.6%·연 1천만원 미만 증여 제외 / 부가가치세법 제32조 세금계산서 / 공증수수료 1천만원=33,000원 정액
  - **연도 의존값(4.6%·수수료)은 작성 시 재확인**
- **출처 링크 1~2개** (본문 자연 삽입): law.go.kr(국가법령정보)·easylaw.go.kr(찾기쉬운 생활법령)·정부24·nts.go.kr(세무)
- **목차형 요약 리스트** (도입부 직후, 4~6개, 각 항목 구체 수치·키워드. "~란?" 추상 항목 금지)
- **마무리 섹션**: 제목 매번 다르게(고정 "마무리" 금지), 4문장+, 핵심 수치 1개 재강조, 폼다 도구 활용 권유로 자연 마무리

## 3. 비주얼 (별도 규칙 = memory project_formda_naverblog)
- 폼다 도구 실캡처 1장(예시 버튼) + SVG 2~3장(내가 생성)
- 제이퍼 연결 글 = 제이퍼 계산기 캡처 1장 대체 / 법률 글 = SVG 인포그래픽 + 선택 정부사이트 캡처

## 4. 폼다 링크·CTA
- 도구 URL `formda.kr/tools/{slug}` (영문 슬러그 실확인, 추정 금지)
- UTM `?utm_source=naverblog_formda&utm_campaign=f{N}&utm_content={top|bottom}`
- 상단 1·하단 1, 자연 연결일 때만
- 원고 팩트: 가입·설치 없음 / 무료(**"워터마크 없음" 표현 금지**) / **유료 1,000원 언급 금지**

## 5. 구조·문체 다양화 (대량생산 강등 회피 = 최우선)
- **작성 유형 5종 중 택1, 직전 2편과 다른 유형**: ①단계 워크스루 ②비교·구분 ③요건·체크리스트 ④전략·관점 ⑤허브·로드맵
- **도입부 6패턴 로테이션**(직전 3편과 다르게): 상황던지기/정의직격/통념반박/질문형/리스크환기/통계수치환기. "안녕하세요 오늘은~" AI 인사 금지
- 본문 구성 순서를 글마다 셔플(시나리오 먼저 / 결론 먼저 / 표 먼저 등), highlight 위치 고정 금지
- "~정리합니다/알아보겠습니다" 전체 1회 이하 / 10자 이하 짧은 문장 섞기 / 필자 판단·의견 1회 / 독자에게 말 거는 톤 섞기
- h2 스타일 다양화: "~하는 법", "~의 진실", "왜 ~인가", "~인 줄 알았는데"

## 6. 주제 선정·팩트 (카니발 2중)
- 온사이트 가이드 34편이 잡은 헤드텀 회피 → 상황 롱테일만 / 블로그 내 중복 회피(직전 제목 grep)
- 동결: 소송절차 상세(로폼)·세무신고 상세(→jptcalc 링크로 넘김)
- 유틸(글자수·로마자·주소·QR·증명사진·PDF) 발행 X, 본문 내부링크만
- 모든 법률·금전 수치는 근거 확보 후 단정, 추측 금지

```
□ 본문 2,500자+ (thin 아님, 패딩 없음)

□ 문서 작성 실사례 3개+ (인물·상황, 대조 2개+, 의사결정 임팩트 문장)
□ 비교·기준표 1~2개 (실제 항목)

□ FAQ 3~5개(직전과 다른 개수), 각 4문장+
□ 신뢰 시그널 2개+ (법조문/고시/통계, 검증수치 정확)

□ 비주얼: 폼다캡처1 + SVG2~3 (또는 제이퍼/정부 캡처)

```
