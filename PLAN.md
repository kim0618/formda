# 폼다 (formda) - 프로젝트 기획서

도메인: **formda.kr** / 이름: **폼다** / 폴더: `formda`
최종 정리: 2026-06-23

---

## 1. 한 줄 정의

**가입 없이 웹에서 입력하면 실무 문서가 바로 완성 PDF로 나오는 한국형 문서 자동작성 도구 사이트.**

핵심 포지션: "양식을 찾는 사이트(서식 도서관)"가 아니라 **"양식을 완성해주는 도구."**

---

## 2. 왜 이 사업인가

- 정보형 콘텐츠 사이트(부모혜택류)는 **AI가 트래픽을 흡수**해서 사양길.
- 살아남는 건 **도구형**: AI는 설명만 하고 실행은 못 하므로, 사용자는 결국 도구로 와야 함.
- jptcalc(계산)·lovtaro(타로)가 가속하는 이유와 동일. 그 패턴을 **문서 자동작성**이라는 새 키워드 우주에 복제.
- 감성/소셜(락룸·Wrapped류)은 네트워크효과 + 유통부재로 전부 기각. 도구형이 제이님 유통(네이버·ChatGPT·블로그)과 맞음.

> 한 줄 원칙: **AI 시대에 살아남는 SEO = AI가 말로 못 끝내고 직접 해줘야 하는 것.**

---

## 3. 포지셔닝 (경쟁사 vs 우리)

**경쟁 = 서식 도서관 (빈 양식 다운로드):** 비즈폼(400만), 예스폼(900만), 프리폼, 바로양식, Jform.
이들의 강점(B2B·물량 14~20만·공신력·HWP편집·head키워드 SEO)은 **못 이김.**

**우리 = 자동작성 도구 (입력→완성):** 그들이 버리는 시장 공략.

| 서식 도서관 (경쟁) | 폼다 (우리) |
|---|---|
| 가입/유료 | 무가입/무료 진입 |
| 빈 양식 다운로드 | 완성 PDF 생성 |
| PC/HWP 중심 | 모바일/웹 중심 |
| 파일 찾아 직접 채움 | 입력폼으로 자동완성 |
| 대량 서식 도서관 | 자주 쓰는 문서에 집중 |

**유일한 차별점 (절대 흐리면 안 됨):** "빈 양식을 주는 게 아니라, 입력하면 완성해준다."
"무가입·무료 다운로드"는 바로양식도 하므로 차별점 아님. **자동작성→완성PDF만이 비어있는 각도.**

키워드 전략: "OO 양식 다운로드"(경쟁 패배) 말고 **"OO 만들기 / 자동작성"** 인텐트 공략.

## 4. 카테고리 구조 (1차)

문서 도메인 기준. **법률문서는 2차로 연기** (법적 리스크 + LawForm/로폼 경쟁 회피).

| 카테고리 | 도구 |
|---|---|
| **① 비즈니스 문서** | 견적서·거래명세서·영수증·발주서·청구서·입금표 |
| **② 취업/자기PR** | 이력서·자기소개서·경력기술서·자소서 글자수 |
| **③ 생활/행정** | 명함·QR코드·바코드·상장/표창장·라벨·가정통신문 |
| **④ 텍스트 도구** | 글자수 세기·여권 영문이름 변환·로마자/주소 변환·정렬 |

**2차에 열 카테고리 (자리만 비워둠):**
- ⑤ 계약/법률 (차용증·내용증명·위임장·근로계약서·임대차) - 면책·구조 정비 후
- 급여명세서 - 4대보험·근로기준법 계산 복잡, 2차

nav 순서: 신규라 GA 없으니 전략 우선순위로 **①비즈니스 → ②취업 → ③생활 → ④텍스트**, 트래픽 쌓이면 GA 기준 재배열.

## 5. 도구 라인업 (우선순위)

### 1차 창끝 (= 견적서 엔진 패밀리, 먼저 제작)
견적서 → 거래명세서 → 영수증 → 발주서 → 청구서.
**같은 컴포넌트(공급자/공급받는자/품목테이블/금액계산/PDF)라 엔진 1개로 5개.**
+ 트래픽 미끼: 글자수 세기, 여권 영문이름 변환.

### 2차
이력서·자기소개서·경력기술서·명함(인쇄제휴)·QR코드·상장·표창장.

### 3차 / 법률 (연기)
차용증(면책)·내용증명·위임장·근로계약서·임대차·급여명세서.

## 6. 화면 구조 (4종 페이지)

모두 공통 셸(sticky 헤더 + 카테고리 nav + 푸터) 위에.

### ① 홈 (한눈에)
- 히어로 1줄 카피 + 검색바
- 🔥 자주 쓰는 도구 8개 카드
- 카테고리별 섹션(비즈니스/취업/생활/텍스트) 전부 스크롤로 조망
- 카드 통일: 동일 크기 + 아이콘 + 이름 (모바일 2열, 데스크톱 4열)

### ② 도구 페이지 (핵심) - 데스크톱
```
홈 > 비즈니스 > 견적서 만들기
┌──────────────┬──────────────────┐
│ [입력 패널]    │ [A4 실시간 미리보기] │
│ 공급자 ____    │   견 적 서         │
│ 품목 [+행추가] │   품목 테이블...    │
│ 부가세[포함▾]  │   합계: ...        │
│ 도장/로고[업로드]│ [PDF 다운로드][인쇄] │
├──────────────┴──────────────────┤
│ 📄 견적서란?·작성법 (정적본문=SEO 프리렌더) │
│ 🔗 관련 도구: 거래명세서·영수증·발주서      │
└─────────────────────────────────┘
```
좌측 입력 → 우측 문서 즉시 채워짐. **미리보기가 주인공.**

### ② 도구 페이지 - 모바일
입력/미리보기 **탭 전환** + 하단 고정 PDF 버튼.

### ③ 카테고리 허브 (SEO)
카테고리 설명(색인용) + 도구 카드 그리드 + 관련 가이드 글 링크.

### ④ 가이드 블로그
본문 + 끝에 "이 문서 바로 만들기 → 도구" CTA. 도구↔글 양방향 내부링크.

## 7. 디자인 원칙

- **컨셉**: 신뢰감 있는 미니멀 업무 도구 (lovtaro 감성의 정반대).
- **색**: 화이트 배경 + 옅은 그레이(#F7F8FA) + 포인트 1색(인디고/블루=신뢰). 화려함 금지.
- **미리보기가 주인공**: A4 문서가 실시간으로 채워지는 게 보여야 전환.
- **마찰 0**: 도구 클릭 → 바로 입력, 가입/온보딩 없음.
- **여백 넉넉**, 카드 크기·간격 통일.
- 타이포 Pretendard, 라운드 8~12px, 토스/노션 톤.

## 8. 기술 구조

- **정적 사이트** (jptcalc 스택 재사용, rsync 배포). 도구는 클라이언트 사이드·무로그인.
- **doc-registry (중요)**: 중앙 데이터 파일 하나에 도구 메타(slug/category/title/keywords/engine) 정의 → 카테고리 nav·허브·사이트맵·내부링크·sibling 자동 생성. **jptcalc 초기 하드코딩 실수 반복 금지, 처음부터 registry.**
- **PDF**: 브라우저 렌더링(window.print + @media print). 한글 안 깨지고 가장 안정적. 도구 UI 숨기고 문서 영역만 인쇄.
- **네이버 색인(프리렌더)**: 도구 페이지에 JS 없이 보이는 정적 본문(도구 설명/작성법 요약) 주입. lovtaro #app 정적 주입 기법 재사용. JS 빈 껍데기로 두면 네이버 색인 안 됨.
- 정적 본문 = 가이드 블로그 요약본 → 콘텐츠 한 번 작성이 도구 색인 + 블로그 둘 다 커버.

## 9. 수익 모델

**무료 완성 + 유료 꾸미기** (다운로드 막으면 이탈, 기본 PDF는 무료).

| 레이어 | 내용 |
|---|---|
| 광고 | 애드핏·애드포스트 (애드센스 추후) |
| 직접결제 | 워터마크 제거·고급 템플릿·로고/도장·거래처 저장·일괄작성 |
| 제휴 | 명함/현수막 인쇄, 세무/노무 (링크프라이스) |

**현실**: 개인 1회성 결제는 약함(팁 수준). **진짜 돈은 소상공인 반복 사용 = 구독(월 4,900~9,900).**

**결제 사다리** (시작은 결제 없이 검증 먼저):
1. 무료 출시 → 트래픽·"워터마크 제거" 클릭 의도 검증
2. **스마트스토어 이용권** (네이버페이, 사업자 가볍게/개인) - 한국 사용자에 Gumroad보다 나음. 단 인앱 즉시결제·자동구독은 약함
3. 규모/UX 필요시 → **토스페이먼츠 인앱 PG** (사업자등록 + 통신판매업신고)

## 10. 유입/성장 전략 (3중 엔진)

도구만 올리면 안 큼. jptcalc 성장 엔진은 블로그였음.

1. **도구 키워드 직격**: 각 도구 페이지가 "OO 만들기"로 직접 랭킹.
2. **작성법 가이드 블로그 (jptcalc 플라이휠 복제)**: 도구당 가이드 3~5편 → 도구로 내부링크. 성장 동력.
3. **북마크 + 카페/커뮤니티 추천**: 무가입·즉시생성 UX → 자영업·취준·맘카페 추천 + 백링크. (경쟁사는 유료라 추천 안 됨 = 우리 강점)

## 11. 개발 순서 & 로드맵

**원칙: 도구 5개를 따로 만들지 말고, 엔진 1개를 만들고 5번 갈아끼운다.**

1. **셋업**: formda.kr, jptcalc 스택, doc-registry 스키마, 디자인 토큰.
2. **견적서 엔진 완성 (진짜 MVP, 1~2주)**: 공통 입력 컴포넌트 + 실시간 A4 미리보기 + 브라우저 PDF + 정적 본문 슬롯.
3. **엔진 복제 (3~5일)**: 거래명세서·영수증·발주서·청구서. → 비즈니스 문서 카테고리 점등.
4. **가이드 콘텐츠 + 내부링크 (지속)**.
5. **무료 출시 + 지표 검증 (1~3개월)**: 결제 0. 색인·유입·PDF 생성률만.
6. **결제 (검증 후)**: 스마트스토어부터.
7. **2차 도구 확장** → 법률 카테고리 오픈.

**타임라인**: 0~3개월 거의 무수익 / 6~12개월 트래픽+첫결제 / 12~24개월 도구 40개+블로그+네이버 크랙 시 jptcalc급 가시권.

**금지**: 도구 10개 동시 벌리기. 견적서 엔진 하나를 제대로 끝내는 게 전부.

## 12. 리스크 / 주의

- **법률문서 = 변호사법 리스크**. 면책만으론 부족. 도구는 포맷만, 내용 100% 사용자 입력, 법률 자문·콘텐츠 자동생성 금지. 1차에서 최대한 뺌.
- **head 키워드(OO 양식)는 경쟁사 독점** → "만들기/자동작성" 인텐트 + UX로만 진입.
- **간이영수증**: 허위 작성 오남용 고지문 필수.
- **여권 영문이름**: 외교부 로마자 표기 규칙 정확도 필요.
- 이건 가벼운 부수입이 아니라 **1년짜리 제2의 jptcalc 프로젝트.**

## 14. 콘텐츠 품질 & 애드센스 정책 (3사이트 교훈 적용)

폼다는 비즈니스/문서 니치라 제이님 사이트 중 **애드센스에 가장 유리** (광고주 친화·단가 높음). 단 도구 사이트 특유의 thin/스캐일 함정을 처음부터 피한다.

**도구 페이지:**
- 각 도구 아래 **고유 본문 800~1,500자** (정의·언제 쓰나·필수항목·작성 예시·주의사항·FAQ)
- 엔진 패밀리라도 **본문 복붙 절대 금지** (견적서·거래명세서·영수증은 용도·항목·예시가 다 다름)
- 프리렌더 본문 = 네이버 색인 + 애드센스 통과 + 사용자 가치 (세 마리 토끼)

**가이드 블로그:**
- 제이퍼 **/blog 스킬 기반** (구조 다양화·AI패턴 검출·내부링크·크롬가드 내장 = 3사 교훈 탑재)
- **질 > 양. AI 양산 금지.**
- 발행: **주 7편 가능**, 단 "숫자가 아니라 주제 가치가 페이스를 정함" — 큐 얇아지면 줄이고 padding 금지
- 주제 확장: 작성법 너머 소상공인 세무·프리랜서 실무·사업자 행정으로 니치 넓힘 (좁은 니치 보완)

**신뢰(E-E-A-T):** 소개·이용약관·개인정보·문의 런칭부터.
**YMYL:** 세무·급여·법률 정확도, 법률 2차 연기.

## 15. 런칭 기준 & 발행 시점

**공개(색인 시작) 시점 - 비교적 일찍, 단 "공사중" 금지:**
- 도구: **법률 제외 전부** (최소 비즈니스 5 + 미끼 2 = 7개, 이상적 12~15개), 각 고유 본문
- 가이드: **10~15편** (한 번에 써서 오픈해도 OK, 단 양질 + 구조 다양화)
- 신뢰 페이지 4종 (소개·약관·개인정보·문의)
- → "실재하는 유용한 사이트"로 보이면 thin/doorway 안 걸림

**애드센스 신청 시점 - 공개 후 3~6개월, 딱 1회:**
- 도구 15개+ 각 고유 본문 + 가이드 25~30편 + 실제 트래픽 + 신뢰 완비

**발행 정책:**
- 런칭: 가이드 15편 프론트로드 OK (한 번에 많이 = 문제 아님, 저품질 양산이 문제)
- 이후: 주 7편 가능 (주제 가치가 페이스 결정, 큐 얇아지면 줄임)
