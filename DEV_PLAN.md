## 모두의문서에서 가져올 것 (벤치마크)

| 가져올 것 | 적용 |
|---|---|
| **"입력 정보는 서버에 저장 안 됨, 모두 브라우저에서 처리"** 명시 | 모든 도구에 트러스트 배지 (개인정보 우려 해소 + 신뢰) |
| **3단계 흐름 (입력 → 실시간 미리보기 → 다운로드)** 명확화 | 홈·도구 페이지에 단계 시각화 |
| **PDF + 이미지(PNG) 둘 다 다운로드** | 우리도 PNG 다운로드 추가 (이미 canvas 캡처하므로 쉬움) |
| **아이콘 카드 그리드 문서 선택** | 홈 디자인에 반영 (이미 계획됨) |
| **재직증명서** (그들 10종 중 하나) | 우리 라인업에 추가 (수요 큼·법적리스크 낮음) → 취업/직장 |
| 누적 생성 통계(사회적 증거) | 런칭 후 숫자 쌓이면 표시 |

우리가 더 나은 점 (차별 유지): 넓이(30+ vs 10) · 견적서 완성도(사업자정보·도장·한글금액) · SEO 콘텐츠 깊이 · 수익 레이어.

## 기술 아키텍처

정적 사이트 + 빌드 스크립트 (제이퍼 calc-registry 방식). 무로그인·클라이언트 처리.

```
formda/
├─ data/registry.js        # 도구 정의 (slug·category·title·keywords·fields·docType)
├─ engine/                 # 공통 모듈 (도구마다 재사용)
│   ├─ form-engine.js      # 입력폼 렌더 + 상태관리
│   ├─ calc.js             # 금액·부가세 계산
│   ├─ doc-render.js       # 문서 미리보기 렌더
│   └─ export.js           # PDF(html2canvas+jsPDF) + PNG 다운로드
├─ templates/              # tool-page · home · category 템플릿
├─ scripts/build.mjs       # registry → 정적 HTML 생성 + sitemap + 내부링크
├─ styles/tokens.css       # 디자인 시스템 (색·간격·컴포넌트)
├─ tools/*.html            # (생성됨) 도구 페이지
├─ guides/*.html           # 가이드 블로그
└─ pages/                  # 소개·이용약관·개인정보·문의
```

핵심 원칙: **도구는 registry 데이터 + 공통 엔진으로 생성.** 새 도구 = 데이터 추가 + 문서 템플릿만.

## 개발 단계

### Phase 0 — 기반 셋업 (3~5일)
- 프로토타입(estimate.html)에서 **공통 엔진 분리** (form/calc/doc-render/export)
- **doc-registry 스키마** 설계 (필드 정의, 문서 타입)
- **디자인 시스템 토큰화** (tokens.css)
- **빌드 스크립트 골격** (registry → 페이지)
- 트러스트 배지("서버 저장 안 함") 공통 컴포넌트

### Phase 1 — 견적서 프로덕션화 (3~4일)
- 프로토타입 → registry 기반 재구성 (엔진 검증)
- **도장 이미지 업로드 + PNG 다운로드** 추가 (모두의문서 참고)
- **모바일 PDF 실측 완료** (1번 구멍 닫기)

### Phase 2 — 비즈니스 엔진 복제 (4~5일)
- 거래명세서 · 영수증 · 발주서 · 청구서 (registry + 문서템플릿만)
- → **비즈니스 카테고리 5종 완성**

### Phase 3 — 사이트 셸 (3~4일)
- 홈(아이콘 카드 그리드 + 3단계 흐름) · 카테고리 허브 · nav · 푸터
- **정적 프리렌더** (도구 본문 색인용) + 내부링크 자동화 (도구↔가이드)
- **신뢰 페이지 4종** (소개·약관·개인정보·문의)
- **서치엔진 등록** (네이버 서치어드바이저 + 구글 서치콘솔) + 사이트맵 제출

### Phase 4 — 나머지 도구 (1~2주)
- 취업/직장: 이력서 · 자기소개서 · 경력기술서 · **재직증명서(추가)** · 사직서
- 생활/행정: 명함 · QR코드 · 바코드 · 상장/표창장 · 가정통신문
- 텍스트: 글자수 세기 · 여권 영문이름 변환 · 로마자/주소 변환 · 텍스트 정렬
- 각 도구 **고유 본문 800~1,500자** (thin 방지)

### Phase 5 — 가이드 콘텐츠 (Phase 2부터 병행)
- 제이퍼 **/blog 스킬**로 도구당 가이드, 런칭까지 **10~15편** (질 우선)

### Phase 6 — 런칭 점검 & 오픈
- 도메인 구매(**formda.kr**) + 배포(rsync, 본인 수행)
- 크롬 가드 · 고아 페이지 체크 · 모바일 QA
- **오픈** (법률 제외 도구 전부 + 본문 + 가이드 15편 + 신뢰 페이지)

### Phase 7 — 런칭 후 (검증 후)
- 결제: 스마트스토어 → 토스 PG (KPI 검증 후)
- 가이드 주 7편 지속 (주제 가치 기준)
- 애드센스: 공개 3~6개월 뒤 1회 (도구 15+·가이드 25~30·트래픽·신뢰 완비)
- 2차: 법률 카테고리(차용증·내용증명·위임장) 면책·구조 정비 후
